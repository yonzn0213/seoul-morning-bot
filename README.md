# 🌅 서울 아침 브리핑 봇

매일 아침 7시, 내가 사는 **서울 자치구**의 ☔ 비 소식과 😷 미세먼지를 텔레그램으로 알려주는 봇입니다.
서버·DB 없이 **GitHub Actions만으로** 무료 운영됩니다.

```

☔ 오늘 비 소식 있어요! (14시~18시, 강수확률 최대 80%) 우산 꼭 챙기세요!
🌡 최저 19°C / 최고 26°C
하늘: 흐림 ☁️

미세먼지(PM10): 45㎍/㎥ · 보통 🔵
초미세먼지(PM2.5): 22㎍/㎥ · 보통 🔵

좋은 하루 보내세요! 💪
```

## 🙋 알림 받는 방법 (유저용)

1. 텔레그램에서 봇 검색: **`@여기에_봇_아이디`** ← (운영자: 본인 봇 아이디로 수정하세요)
2. **/start** 전송
3. 잠시 후(최대 10분) 도착하는 버튼에서 **내 지역(구) 선택** — 끝!
4. 다음 날 아침 7시부터 매일 브리핑이 도착합니다.

| 명령어 | 기능 |
|--------|------|
| `/start` | 알림 시작 / 지역 선택 |
| `/region` | 지역 변경 (한 번 고르면 계속 유지) |
| `/stop` | 알림 해지 |

> ⏱ 봇이 서버 없이 동작해서 등록 응답이 **최대 10분** 걸릴 수 있어요. 등록은 한 번뿐이니 양해를!

## 🛠 직접 운영하기 (배포 가이드)

나만의 봇을 띄우고 싶다면 아래를 따라 하세요. **15~20분**이면 됩니다.
필요한 것: GitHub 계정, 텔레그램, 공공데이터포털 계정 (전부 무료)

### 1단계. 공공데이터포털 API 키 발급

1. **https://www.data.go.kr** 접속 → 우측 상단 **회원가입** (네이버/카카오 간편가입 가능) → 로그인
2. 검색창에 **`기상청_단기예보`** 검색
3. 결과에서 **「기상청_단기예보 ((구)_동네예보) 조회서비스」** 클릭 (오픈 API 탭)
4. **[활용신청]** 버튼 클릭
   - 활용목적: **웹 사이트 개발** 또는 **앱개발** 선택, 내용에 "텔레그램 날씨 알림 봇" 정도로 입력
   - 상세기능정보: **전체 체크**
   - 라이선스 표시 동의 체크 → **신청**
   - ✅ 자동승인이라 **즉시 사용 가능**합니다
5. 같은 방법으로 **`에어코리아_대기오염정보`** 검색 → **「한국환경공단_에어코리아_대기오염정보」** → **[활용신청]**
6. **마이페이지 → 데이터 활용 → Open API → 활용신청 현황** → 방금 신청한 API 클릭
7. **일반 인증키 (Decoding)** 항목의 키를 복사해두세요
   - ⚠️ **Encoding 키가 아니라 Decoding 키**입니다! (코드가 알아서 인코딩함)
   - 인증키는 **계정당 1개**라서 두 API에 같은 키를 씁니다
8. 💡 발급 직후 30분~1시간은 키 동기화 지연으로 `SERVICE_KEY_IS_NOT_REGISTERED` 오류가
   날 수 있어요. 그럴 땐 잠시 후 다시 시도하면 됩니다.

### 2단계. 텔레그램 봇 만들기

1. 텔레그램에서 **@BotFather** 검색 → 대화 시작
2. `/newbot` 전송 → 봇 이름 입력 (예: `서울 아침 브리핑`) → 봇 아이디 입력 (예: `seoul_briefing_bot`, `bot`으로 끝나야 함)
3. 발급되는 **봇 토큰** 복사 (예: `1234567890:AAH...`)
4. (추천) `/setcommands` → 봇 선택 → 아래 붙여넣기 (메뉴에 명령어가 떠서 쓰기 편해짐):
   ```
   start - 알림 시작 / 지역 선택
   region - 지역 변경
   stop - 알림 해지
   ```

### 3단계. GitHub 저장소 만들기

1. 이 저장소 우측 상단 **Fork** (또는 이 폴더를 새 public 저장소에 push)
2. 내 저장소 → **Settings → Secrets and variables → Actions → New repository secret** 으로 2개 등록:

   | Name | Value |
   |------|-------|
   | `DATA_GO_KR_KEY` | 1단계의 공공데이터포털 **Decoding** 인증키 |
   | `TELEGRAM_BOT_TOKEN` | 2단계의 봇 토큰 |

3. **Settings → Actions → General → Workflow permissions** 에서
   **Read and write permissions** 선택 후 Save (유저 목록 자동 커밋에 필요)
4. **Actions 탭** → 배너가 뜨면 **"I understand... enable them"** 클릭해 워크플로 활성화
   (Fork한 경우 스케줄 워크플로가 기본 비활성화되어 있습니다)

### 4단계. 동작 테스트

1. 내 봇에게 텔레그램으로 **/start** 전송
2. **Actions 탭 → 「유저 등록 처리」 → Run workflow** (기다리기 싫을 때 수동 실행)
3. 지역 버튼 도착 → 내 구 선택
4. **Actions 탭 → 「서울 아침 브리핑」 → Run workflow** → 브리핑 도착하면 성공! 🎉
5. README 상단의 봇 아이디를 본인 봇으로 수정해서 친구들에게 공유하세요.

## ⚙️ 구조

| 파일 | 역할 |
|------|------|
| `register.py` | 10분마다 새 메시지 폴링 → 등록/지역변경/해지 처리 |
| `briefing.py` | 매일 아침 유저별 지역 브리핑 전송 |
| `common.py` | 서울 25개 구 기상청 격자좌표, 텔레그램 헬퍼, 상태 저장 |
| `state.json` | 유저 목록 (Actions가 자동 커밋 — DB 불필요) |
| `.github/workflows/` | 등록 폴링(10분) + 일일 브리핑(KST 07:00) 스케줄 |

- 날씨: 기상청 단기예보 API — 구별 격자좌표, 새벽 5시 발표분 사용
- 미세먼지: 에어코리아 API — 구별 측정소 실측값, 측정소 점검 시 서울 평균으로 자동 대체

## 📝 운영 참고사항

- **공개 정보**: public 저장소라 `state.json`의 유저 chat_id 목록이 공개됩니다.
  chat_id만으로는 메시지를 보낼 수 없고(봇 토큰 필요) 토큰은 Secrets에 안전하게 보관되지만,
  이 점이 부담되면 private 저장소로 바꾸고 `register.yml` cron을 `*/30 * * * *`로 조정하세요 (무료 한도 월 2,000분).
- **발송 시간 변경**: `daily-briefing.yml`의 cron은 **UTC** 기준 (KST−9시간). KST 6:30 → `30 21 * * *`
- **cron 지연**: GitHub 무료 러너 특성상 수~수십 분 지연될 수 있습니다. 중요하면 앞당겨 설정하세요.
- **60일 규칙**: 저장소에 60일간 커밋이 없으면 GitHub이 스케줄을 자동 정지합니다.
  유저 활동이 있으면 state.json 커밋이 생겨 보통 문제없고, 정지 알림 메일이 오면 Actions 탭에서 재활성화하면 됩니다.
- **API 한도**: 공공데이터포털 개발계정은 일 1,000건 안팎(API별 상이) — 이 봇 사용량으론 충분합니다.
- **다른 지역 확장**: `common.py`의 `REGIONS`에 격자좌표 추가 + `briefing.py`의 `sidoName` 수정
